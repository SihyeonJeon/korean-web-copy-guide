# KOREAN_SERVICE_PAGE_PATTERNS.md

한국어 서비스 웹페이지 카피 패턴 팩.

`KOREAN_WEB_COPY.md`를 기본 규칙으로 사용하고, 이 파일은 실제 한국어 서비스 페이지의 구조와 말투를 참고해 랜딩 페이지, 기능 페이지, 요금 페이지, 사례 페이지 문구를 만드는 데 사용한다.

이 파일은 출처 문구를 재사용하지 않는다. 제품 사실은 새 프로젝트의 brief에서만 가져온다. 여기서 가져올 것은 문장 구조, 정보 배치, CTA 형태, 문제 제기 방식, 문장부호 사용 규칙이다.

## 전체 관찰 요약

한국어 서비스 페이지는 긴 설명보다 첫 화면에서 다음 네 가지를 빠르게 보여준다.

1. 누구의 어떤 작업인가
2. 기존 방식의 불편은 무엇인가
3. 이 서비스가 어떤 흐름을 한곳에 묶는가
4. 바로 할 수 있는 행동은 무엇인가

반복되는 구조:

```text
[대상/도메인]을 위한 [제품 범주]
[기존 방식/반복 작업]에서 벗어나 [핵심 결과]를 [시간/상태/흐름] 기준으로 확인
[무료 체험] [도입 문의] [소개서 다운로드]
```

많이 쓰이는 방향:

- `한 번에`
- `바로`
- `쉽게`
- `간편하게`
- `놓치지 않고`
- `실시간`
- `자동`
- `무료로 시작`
- `도입 문의`
- `더 알아보기`
- `소개서 다운로드`

주의:

- 위 단어만 붙이면 얕아진다
- 반드시 실제 대상, 입력, 처리 기준, 출력과 함께 쓴다
- `쉽게`, `간편하게`, `한 번에`는 뒤에 구체 작업을 붙인다


```text
현장 작업팀을 위한 근무 기록 관리
출퇴근, 작업 배정, 계약 서명을 프로젝트별로 정리
```

### 3. 요금 페이지

권장 순서:

1. 무료/시작 플랜
2. 팀용 플랜
3. 확장/엔터프라이즈 플랜
4. 과금 기준
5. 포함 기능
6. 제한 조건
7. 상담 CTA

문장형:

```text

```

문구 규칙:

- `문의` 플랜은 "맞춤 견적"보다 "도입 환경 확인"이 자연스럽다

### 4. 사례 페이지

권장 순서:

1. 고객 유형
2. 도입 전 문제
3. 바뀐 작업 흐름
4. 관찰 가능한 결과
5. 사용 기능
6. 짧은 인터뷰형 문장

문장형:

```text

```

주의:

- 고객이 실제로 말하지 않은 문장을 인용문처럼 만들지 않는다
- 성과 수치가 없으면 수치처럼 보이게 쓰지 않는다
- `만족`, `성공`, `압도적` 같은 평가는 고객 인터뷰나 지표가 있을 때만 쓴다

### 문제 제기

한국어 서비스 페이지는 문제를 너무 추상적으로 말하지 않는다. 오래된 도구, 반복 작업, 놓치는 일정, 담당자 혼선, 규정 리스크처럼 실제 장면을 먼저 둔다.

문장형:

```text
아직도 [기존 방식]으로 [업무]하고 있나요?
```

```text
[업무 A]와 [업무 B]가 따로 관리되고 있나요?
```

```text
[중요 상태]를 놓치고 있지 않나요?
```

```text
[작업]보다 [정리/보고/확인]에 시간이 더 쓰이고 있나요?
```

주의:

- 한 페이지에 질문형을 너무 많이 쓰지 않는다
- 엔터프라이즈/법무/인사 페이지에서는 감탄형보다 리스크·기준·상태를 쓴다
- 소비자형/마케팅형 페이지에서는 가벼운 질문과 감탄형이 허용된다

### CTA

기본 CTA:

- `무료로 시작하기`
- `도입 문의`
- `상담 신청`
- `데모 보기`
- `소개서 다운로드`
- `가격 확인`
- `더 알아보기`

CTA 선택 기준:

| 상황 | 1차 CTA | 2차 CTA |
| --- | --- | --- |
| 셀프서브 SaaS | 무료로 시작하기 | 가격 확인 |
| B2B 도입형 | 도입 문의 | 소개서 다운로드 |
| 기술/API 제품 | 문서 보기 | 상담 신청 |
| 마케팅/크리에이터 도구 | 무료 체험 | 사례 보기 |
| 엔터프라이즈/현장 운영 | 상담 신청 | 기능 보기 |
| 법무/인사/리스크 | 도입 문의 | 자료 받기 |

CTA 규칙:

- `시작하기`만 쓰지 말고 무엇을 시작하는지 주변 문구가 설명해야 한다
- `문의하기`보다 `도입 문의`, `상담 신청`이 구체적이다
- `무료`는 실제 무료 플랜/체험이 있을 때만 사용한다
- `더 알아보기`는 보조 CTA로 둔다

### 문장부호

권장:

- `·`: 기능 나열, 제품 범위, 채널 나열
- `|`: 메타 제목 또는 문서 제목
- `,`: 대상과 범주 분리
- `?`: 문제 제기 섹션에서 제한적으로 사용
- `!`: 프로모션, 가벼운 B2C/마케팅형 화면에서 제한적으로 사용
- `→`: 링크성 보조 CTA 또는 자료 이동

회피:

- 모든 카드 끝에 마침표
- 제목에 긴 쉼표 나열
- 한 화면에서 `!`와 `?` 남발
- 이모지로 근거 없는 흥분 만들기

이모지:

- 이벤트, 무료 체험, 가벼운 마케팅/뷰티/커머스 톤에서만 허용
- 법무, 인사, 보안, 엔터프라이즈 페이지에서는 기본적으로 사용하지 않음

8. 출처 서비스명과 고유 문구 제거
9. 마침표, 과장어, 설명문 주어 점검
